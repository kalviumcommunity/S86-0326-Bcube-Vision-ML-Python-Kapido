================================================================================
  COMMON WORKFLOWS
================================================================================

Daily Development:
──────────────────
  $ activate_env.bat                    # Activate venv
  $ python -m src.main                  # Train model
  $ pytest src/test_pipeline.py -v      # Run tests
  $ deactivate                          # Exit venv

Add New Dependency:
──────────────────
  $ activate_env.bat
  $ pip install new_package
  $ pip freeze > requirements.txt
  $ git add requirements.txt
  $ git commit -m "Add new_package"

Share with Team Member:
────────────────────────
  1. They clone the repo
  2. They run: python -m venv venv
  3. They run: activate_env.bat (or source venv/bin/activate)
  4. They run: pip install -r requirements.txt
  5. They have identical environment!

Deploy to Production:
─────────────────────
  $ python -m venv venv
  $ venv\Scripts\activate
  $ pip install -r requirements-frozen.txt
  $ python -m src.main

================================================================================
  IMPORTANT POINTS
================================================================================

✅ DO:
   • Activate venv before running code
   • Update requirements.txt when adding packages
   • Commit requirements.txt to git
   • Use pip freeze for exact versions
   • Check (venv) in prompt to confirm activation

❌ DON'T:
   • Commit venv/ to git (it's in .gitignore)
   • Delete venv/ during development
   • Use global pip when venv is activated
   • Hardcode paths that assume venv/ location
   • Activate multiple venvs at once in same terminal

================================================================================
  TROUBLESHOOTING
================================================================================

Problem: "venv not found"
Solution: Run the commands from project root directory
          $ cd S86-0326-Bcube-Vision-ML-Python-Kapido
          $ activate_env.bat

Problem: "(venv) not in prompt"
Solution: Virtual environment not activated
          Try: venv\Scripts\activate (Windows)
          Or:  source venv/bin/activate (Unix)

Problem: "pip: command not found"
Solution: Activate venv first or use:
          $ python -m pip (instead of just "pip")

Problem: "ModuleNotFoundError: No module named 'sklearn'"
Solution: Install packages or activate venv:
          $ pip install -r requirements.txt

Problem: "Permission denied" (macOS/Linux)
Solution: Make scripts executable:
          $ chmod +x venv/bin/activate
          $ source venv/bin/activate
